## EDITORIAL & BUSINESS STYLES

### Modern Newspaper Style
```
You are a top art director leading Japan's "new economy business media." 

Design Philosophy:
- Swiss Style (International Typographic Style) or Bauhaus
- Asymmetrical layout, grid-based positioning
- Ban centered alignment
- Title placed top-left or bottom-left

Special Specifications:
- Complete exclusion of Markdown symbols (no #, *, **)
- Plain text only on slides
- Main title (2-5 characters, ultra-large)
- Subtitle (concise, benefit-driven)

Overall Design Definition:
1. Core Theme: Smart & pop business infotainment
2. Color Palette:
   - Background: White (#FFFFFF) or Cool Gray (#F5F5F5)
   - Text: Sumi/Jet Black (#111111)
   - Accent: Electric Yellow (#FFCC00) or Alert Red (#FF3333)
3. Visual Style:
   - Smartphone-first economic media aesthetic
   - Monochrome cutouts of people
   - Blown-out background photos
   - Fluorescent marker-style highlights (yellow background)
4. Typography:
   - Headlines at ultra-massive size (30-50% of slide)
   - Extreme jump ratio: 10:1 (headline to body)
   - Extra-bold sans-serif for headlines
   - Ultra-thin English fonts
5. Overall Structure:
   - 1 slide = 1 message
   - Binary choice: negative space OR text
   - Conclusion with visual impact
```

```
Overall Style: Neo-Retro Dev Deck / Pixel-Infographic Editorial

   Aesthetic: Retro-futuristic, developer-centric, editorial infographic style
   Feels like: 90s computer manuals + modern AI dev tools + pixel-art startup deck

2. Canvas & Background
   Use light cream/off-white grid paper background
   Subtle square grid (engineering notebook feel)
   Grid lines very light, never distracting

   Primary headline: Bold, heavy sans-serif, slightly condensed
   Language mixing: Headline language + English sub-labels (small, clean)
   Hierarchy: Huge bold headline blocks > Medium sub-titles > Small captions

4. Color Palette (Strict)
   Use high-contrast blocks with thick black outlines:
   - Hot Pink (agent/brain/intelligence)
   - Bright Yellow (editor/code/tools)
   - Cyan/Light Blue (browser/web/execution)
   - Black (text, borders)
   - White/cream (background)

5. Layout Language
   Stacked modular blocks
   Rectangles with thick black borders
   Slight overlaps (intentional, collage-like)
   Horizontal bars for section headers
   Controlled imperfection

6. Iconography & Graphics
   Pixel-art style icons (8-bit or 16-bit inspired)
   Flat colors, black outline
   Decorative elements: Small gears, arrows, chevrons, pixel sparks

7. Content Patterns
   A. System Architecture: Stacked layers with labels
   B. Evolution/Timeline: Left→right progression, each in box
   C. Manifesto/Thesis: Huge headline in frame, minimal text, playful icons

8. Tone of Text
   Short, declarative, slightly opinionated
   No marketing fluff
```

## CORE PRESENTATION GENERATION PROMPTS

### Basic Slide Deck Generation
```
"Generate a 7-slide presentation summarizing the key points from all sources 
with titles, content points, and speaker notes."

"Create a presentation deck introducing the top 5 trends from these articles, 
including a conclusion slide and talking points."

"Summarize this PDF into 5 key slides for a beginner audience."

"Extract main arguments and counterarguments for a debate and format them as 
presentation slides."
```

### Idea Extraction (Phase 1)
```
"Read all sources and identify the 7–10 most important ideas that should be in 
a presentation. For each idea, suggest: 
(1) a slide title (6 words max), 
(2) the key evidence or example to include, and 
(3) what the audience should understand or feel after seeing this slide."
```

### Narrative Structure (Phase 2)
```
"Structure these extracted ideas into a presentation flow: opening hook → 
problem/context framing → key insights (3–5) → implications → call to action. 
Each section should connect to the next through explicit transitions, creating 
a deck that tells a story rather than listing facts."
```

### Slide Specifications (Phase 3)
```
"Produce detailed slide-by-slide specs for each key idea: 
- Slide title (6 words max)
- Key content points
- Supporting evidence from sources
- Visual direction (chart type, image suggestion, layout)
- The one thing the audience should understand after seeing this slide"
```

### Speaker Notes & Q&A Prep (Phase 4)
```
"Generate speaker notes for every slide — not a verbatim script, but:
- Key talking points
- Transition cues
- Timing estimates

Then run the Q&A preparation prompt to anticipate the 5 most likely audience 
questions and draft concise, evidence-backed answers."
```

### Audience Versioning (Phase 5)
```
"Take this core deck and adapt it for different audiences:
- Technical vs. executive
- Internal vs. external
- 10-minute lightning talk vs. 45-minute keynote

Adjust depth, vocabulary, emphasis, and slide count while keeping core message intact."
```

## ADVANCED WORKFLOW PROMPTS

### Multi-Format Versioning
```
"Create three versions of this deck:

1. Executive Version (8 slides)
   - Key findings only
   - Business language
   - Decision-focused

2. Detailed Version (20 slides)
   - Full methodology
   - All supporting data
   - Technical language

3. Public Version (15 slides)
   - Customer-friendly
   - Simplified concepts
   - Visual emphasis"
```

### Narrative Arc Construction
```
"Structure this information using the hero's journey:
1. The ordinary world (current state)
2. The call to adventure (the problem/opportunity)
3. Crossing the threshold (the insight/solution)
4. Tests and allies (proof points)
5. The ordeal (challenges overcome)
6. The reward (benefits/results)
7. Return home (implementation/next steps)"
```

### Visual Consistency Branding
```
"Build visual consistency across all slides:
1. Use a consistent color palette: [colors]
2. Apply typography hierarchy: [font sizes/weights]
3. Create visual metaphors that unify the narrative
4. Use consistent spacing and alignment
5. Place brand elements consistently"
```

### Citation & Source Grounding
```
"For every claim on every slide, include:
- The source document it comes from
- The specific quote or data point
- Page reference if applicable

This ensures presentations are defensible and credible."
```

### Progressive Disclosure Strategy
```
"Create a slide deck that uses progressive disclosure:
- Slide 1: The headline (the big idea)
- Slides 2-3: Context and problem definition
- Slides 4-7: Evidence and supporting details
- Slides 8-9: Implications and recommendations
- Slide 10: Call to action

Each slide builds on the previous, revealing complexity gradually."
```

### Visual Metaphor Development
```
"For each key concept, develop a visual metaphor:
- Identify the core concept
- Find a real-world parallel
- Illustrate the parallel visually
- Connect back to the concept
- Make the metaphor consistent across related slides"
```

### Stakeholder-Specific Customization
```
"Create stakeholder-specific versions:

For Finance:
- Lead with ROI and financial impact
- Use numbers prominently
- Clear cost-benefit analysis

For Engineering:
- Emphasize technical detail and feasibility
- Show architecture and implementation
- Highlight technical achievements

For Marketing:
- Focus on customer benefits
- Use customer testimonials
- Emphasize market opportunity

For Operations:
- Clear timeline and milestones
- Resource requirements
- Implementation roadmap"
```

### Counter-Argument Integration
```
"For a persuasive deck, include:
1. Your main argument (slides 1-3)
2. Anticipated objections (slide 4)
3. Counter-evidence you've considered (slide 5)
4. Why your argument still stands (slide 6)
5. Strongest supporting evidence (slides 7-8)
6. Call to action (slide 9)

This builds credibility by acknowledging opposing viewpoints."
```

### Visual Hierarchy Optimization
```
"Optimize visual hierarchy on each slide:

1. Primary Element: The most important insight (dominant size/position)
2. Secondary Elements: Supporting details (medium size/emphasis)
3. Tertiary Elements: Contextual information (small/light emphasis)
4. Background: Subtle visual foundation (very light/neutral)

Use size, color saturation, and positioning to guide the eye."
```

### Story-Driven Structure
```
"Structure the presentation as a story:

ACT 1 - Setup (slides 1-3)
- Who are we talking to? (audience)
- What's their current situation? (context)
- What's the problem? (conflict)

ACT 2 - Confrontation (slides 4-7)
- What did we discover? (plot development)
- What does this mean? (complications)
- What's at stake? (rising action)

ACT 3 - Resolution (slides 8-10)
- What's our solution? (climax)
- How do we implement it? (falling action)
- What happens next? (resolution)"
```

## PROMPT COMPOSITION TIPS

### For Maximum Effectiveness:

1. **Be Specific**
   - Mention exact number of slides
   - Specify target audience
   - Indicate any particular style or format
   - Include tone/mood descriptors

2. **Leverage Suggested Questions**
   - NotebookLM suggests follow-up prompts after uploads
   - Use them to dig deeper or clarify results
   - Ask iteratively to refine outputs

3. **Refine Iteratively**
   - Don't settle for first result
   - If output needs adjusting, refine prompt
   - Ask follow-up questions
   - Build on previous outputs

4. **Ask for Visuals**
   - Request charts or diagrams summarized
   - Mention specific visualization types
   - Describe visual style preferences
   - Ask for infographic-style layouts

5. **Push Multimodal Content**
   - Upload images or charts
   - NotebookLM analyzes and cites them
   - References them in speaker notes
   - Incorporates them in visual suggestions

6. **Include Context Clues**
   - Mention industry/field
   - Reference similar presentations (if available)
   - Specify regulatory or audience constraints
   - Note any brand guidelines
